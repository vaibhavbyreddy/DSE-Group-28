\chapter{Requirements and constraints}
Having evaluated the system functions and resources, the requirements of the product can be fully specified. To that end, a requirement discovery tree is created in \autoref{sec: discovery tree}.


\section{Requirement Discovery Tree}
\label{sec: discovery tree}
In order to identify the system requirements based on the user needs and project objectives, a requirement discovery tree has been generated (\autoref{fig:req_discovery_tree}). The tree starts with the requirement for designing a flying model for testing flexible wings, which is split into the requirement to perform the flight test and the requirement to satisfy the constraints. The functional requirements stem directly from the functions that the design has to be able to accomplish, as stated in the Functional Breakdown.

The constraints are split into complying with budgets (mass, payload volume, power, internal volume, communication link budget, van volume, cost, payload aerodynamic, c.g.), airworthiness requirements, manufacturability, lifetime, sustainability and UAV capabilities, and withstanding the aerodynamic loads.
